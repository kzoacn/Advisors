# AGENTS.md

## 项目摘要

Advisors 是一个中国高校教师公开事实数据集项目。当前 v0.1 阶段采集清华大学、北京大学和 C9 高校的高校官网教师主页信息，核心输出为版本化 Parquet 数据包。

教师主页缓存是基础事实信息源之一。采集流程应先缓存教师主页，再从缓存抽取结构化数据。

目录约定：每所学校单独一个目录。缓存放在 `data/cache/<school>/`，中间产物和 release 放在 `data/working/<school>/`，跨校汇总文件放在 `data/working/` 根目录。

## 当前边界

- 只使用高校官网、院系官网、实验室官网、官方教师主页系统等官方公开页面。
- v0.1 尽量覆盖清华大学、北京大学和 C9 高校所有公开教师主页。
- 用覆盖率台账说明“尽量覆盖”的口径，包括入口、列表页、主页、缓存页、解析页和失败原因。
- 暂不使用 OpenAlex、CNKI、万方、维普、专利库、外部基金查询库。
- 基金、项目、论文、专利、荣誉等内容如果出现在教师官网主页中，先以文本形式保存。
- 合作网络以后生成，不属于第一阶段事实数据。
- 原始 HTML 或等价页面缓存默认不进入公开 release，除非后续明确评估版权、隐私和体积策略。
- 只抓取白名单官方域名。外跳个人站、商业平台、GitHub、Google Scholar、ResearchGate、LinkedIn 等只记录 URL，不采集正文。

## 隐私规则

永不入库任何电话号码，包括手机号、办公室电话、固定电话、传真等。

永不入库身份证号、护照号、证件号、家庭住址。

出生日期、电子邮箱、办公地址等信息可以入库，但必须来自高校官网公开页面，并保留来源 URL。

## 数据要求

- 每条记录必须可追溯到 source_url。
- 每条记录应包含 fetched_at。
- 采集代码应保留页面 hash 或等价的内容版本标识。
- 维护教师主页缓存，缓存应包含 URL、采集时间、内容 hash、状态码和本地缓存路径等复核信息。
- 缓存目录默认不进入 git，也默认不进入公开 release。
- 结构化抽取应从缓存读取，避免在线抓取和字段解析耦合。
- `person_id` 在 v0.1 只是暂定融合结果，应配合 `person_id_status` 和 `identity_confidence` 使用。
- 输出以 Parquet 为核心格式。
- 每个 release 应包含 `manifest.json`，记录版本、schema、生成时间、源码 commit、文件清单、行数、来源数量、缓存批次和报告路径。
- 不做导师评价、排名、推荐分等主观字段。

## 爬取礼仪

- 遵守 robots.txt 和公开访问限制。
- 使用清晰、克制的 User-Agent。
- 设置请求限速、重试上限和退避策略。
- 不绕过登录、验证码、IP 限制、访问控制或反爬措施。
- 不采集需要认证后才能访问的内容。

## Subagent 要求

从事实信息抓取到结构化数据的流程总是开启 subagent 完成。

主流程负责范围、缓存策略、schema 和质量标准；subagent 负责或审查从缓存页面到结构化记录的抽取、字段清洗、排除字段检查和质量报告。

尤其需要 subagent 检查电话号码、证件号、家庭住址等排除字段没有进入任何结构化输出。
